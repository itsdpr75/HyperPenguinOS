What makes it different

HyperPenguinOS rethinks Linux from the ground up. No package manager, no /home/, no X11 legacy.

AreaTraditional LinuxHyperPenguinOS
Package Management apt, pacman, dnf, etc. None native. AppImage, Flatpak, Linyaps via hpk
Cross-distro apps Not supported natively hbox: run .deb/.rpm in Podman containers, integrated into desktop
User filesystem /home/{user}/ + ~/.config/ + ~/.local/ Everything under /user/{user}/: config/, home/, apps/
System updates Package-by-package (risk of partial upgrades) Atomic A/B: whole-image updates, instant rollback
Immutability Mutable root (unless explicitly configured) /usr and / read-only at runtime by default
Rollback Manual snapshots or not available Built-in KCM in Plasma System Settings, CLI script
Display server X11 (legacy) + optional Wayland Wayland native, no X11 (no XWayland in v1)
Bootloader Single entry Dual-entry with automatic fallback on failure
Build system Binary packages from distro repos Built entirely from source with LFS, checkpoint-based recovery

Key features

Every design decision serves one goal: a simpler, safer, more maintainable Linux.

Immutable A/B System

Two root subvolumes (@rootfs-A and @rootfs-B). Updates write to the inactive side. If the new boot fails, GRUB automatically falls back. Atomic by design.

/user/ Filesystem

All user data unified under /user/{username}/: config/, home/, apps/. One directory to back up, one to migrate, no XDG scattering.

hpk — Unified App Manager

Single CLI to install, search, run and remove AppImages, Flatpaks, Linyaps and containerized apps. Auto-detects format, integrates with KDE Discover.

hbox — Cross-Distro Containers

Run Ubuntu, Fedora or Arch containers via Podman. Install .deb/.rpm packages inside and get .desktop files generated. Wayland, PulseAudio and GPU mounted automatically.

KCM Rollback

Native Plasma System Settings module to browse and restore BTRFS snapshots. Full integration with the A/B update mechanism for instant system recovery.

Wayland Native

KDE Plasma 6 on Wayland exclusively. No X11, no XWayland in v1. AMDGPU + Vulkan via Mesa. Cleaner, faster, and more secure display stack.

Architecture

From the kernel up to the desktop, every layer is built from source and designed for immutability.

Tools

Two CLI tools and a Plasma module that replace an entire ecosystem of package managers.

hpk — HyperPenguin Kommander

Unified application manager. Install anything from anywhere with one command — AppImage, Flatpak, Linyaps, or containerized via hbox.

# Auto-detect format hpk install firefox # Force a format hpk install --flatpak org.gimp.GIMP hpk install --box ubuntu firefox # List, search, run hpk list hpk search firefox hpk run org.gimp.GIMP

hbox — Cross-Distro Containers

Run applications from other distributions inside Podman containers. Transparent desktop integration with generated .desktop files.

# Create containers hbox create ubuntu --image ubuntu:24.04 hbox create fedora --image fedora:40 # Install & run apps hbox install ubuntu firefox hbox run ubuntu firefox # Search across all containers hbox search firefox

KCM Rollback

A Plasma System Settings module that lists available BTRFS snapshots and lets you restore any of them with one click. Integrated with the A/B mechanism — snapshots are taken before each update and before each rollback, so you never lose the ability to go back.

# The equivalent CLI (penguin-rollback) also exists manually: penguin-rollback # Lists snapshots, prompts for an ID, restores it, # backs up the current state automatically.

System Scripts

Three CLI tools manage the A/B lifecycle:

penguin-update # Download + verify + apply atomic update to inactive partition penguin-rollback # List + restore BTRFS snapshots with automatic backup penguin-firstboot# Initial setup: hostname, locales, GRUB, initial snapshot
